## 设备数量限制原理（核心功能）

设备数量限制是订阅管理的核心功能，详细原理请参考：

- [订阅列表管理 - 设备数量限制原理](./subscription_list_management.md#设备数量限制原理)
- [设备管理 - 设备数量限制原理](./device_management.md#设备数量限制原理)

### 关键概念

1. **设备识别**
   - 基于 User-Agent 和 IP 地址生成设备哈希
   - 相同设备和软件版本生成相同哈希

2. **设备漫游**
   - 同一设备在不同网络环境下自动识别
   - 避免因IP变化被重复计算

3. **限制机制**
   - 设备未超限：新设备可以添加
   - 设备达到限制：新设备无法添加，已存在设备可用
   - 设备超过限制：采用"最近使用优先"策略

4. **优先级策略**
   - 按最后访问时间排序
   - 取前 N 个设备作为允许的设备
   - 长时间未使用的设备自动淘汰
